String.split()结尾空字符串将被丢弃

发表于 2017-07-03 | 更新于 2019-05-06 | 分类于 后端开发

测试

String str=”1#2#3############”;
String[] split = str.split(“#”);
for (String string : split) {
System.out.println(“str “+string);

测试结果

str 1
str 2
str 3

问题：#号中间的空无法获取？（官方解释） split(String regex, int limit)方法，头一个参数String regex表示字符串分割的模式，包括分隔符和正则表达式；但是第二个参数limit比较迷糊人，api中这样解释： limit 参数控制模式应用的次数，因此影响所得数组的长度。如果该限制 n 大于 0，则模式将被最多应用 n - 1 次，数组的长度将不会大于 n，而且数组的最后一项将包含所有超出最后匹配的定界符的输入。如果 n 为非正，那么模式将被应用尽可能多的次数，而且数组可以是任何长度。如果 n 为 0，那么模式将被应用尽可能多的次数，数组可以是任何长度，并且结尾空字符串将被丢弃。 解决 将String[] split = str.split(“#”);改为String[] split = str.split(“#”，-1);（其实第二个参数只要是负数就可以）
