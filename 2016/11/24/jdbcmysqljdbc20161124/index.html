JDBC链接Mysql数据库是遇到的问题

发表于 2016-11-24 | 更新于 2019-05-06 | 分类于 Bug

第一个问题是 在连接数据库的时候一直连接不上,报出 Caused by: java.sql.SQLException: Access denied for user ‘minjj‘@’localhost’ (using password: YES) 的错误,但是数据库连接账号应该是root, 可能是在配置文件直接配置了username=root,spring读取配置文件时,${username}可能读到了电脑的管理员名称产生了冲突. 只需要将配置文件中username稍作修改就连接成功了. 第二个是 上面的问题解决之后又出现了 The server time zone value ‘�й���׼ʱ��’ is unrecognized or represents more than one time zone. 这样的错误. 这个错误的原因是mysql jdbc驱动包过新的原因,jar包版本是6.0.3的, 只需要在数据库连接url加上serverTimezone =UTC即可

db.url=jdbc:mysql://127.0.0.1:3306/mssm?useUnicode=true&characterEncoding=utf8&serverTimezone=UTC

后来在测试的时候还遇到一个诡异的问题，就是数据库中的时间比如是：2016-11-11 12:00:00 但是通过程序查出来的时间一直是 2016-11-11 20:00:00 也就是比数据库中时间快了 8个小时，查了很久的相关问题，都说是mysql 时区的问题，但是查我的mysql时区并没有什么问题，也尝试一些方法去解决，都没有什么效果。最后还是将 mysql 驱动包从6.0.3 版本换成了 5.1.25版本的，问题竟然就没有了。╮(╯▽╰)╭
