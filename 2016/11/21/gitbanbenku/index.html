Git-版本库

发表于 2016-11-21 | 更新于 2019-05-06 | 分类于 Git

在git学习之前，需要先安装git,因为用的是windows环境，百度git,傻瓜式安装，安装完成后会发现右键菜单多了git gui here,以及git base here 俩个选项。 版本库就是仓库，英文名repository，其实一个文件夹，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以“还原”。

创建版本库

创建版本库也很容易，只需要选择你想创建版本库的文件夹，输入命令git init 就可以将这个目录变成一个git版本库。

$ git init;
Initialized empty Git repository in G:/project/.git/

版本库创建完成后，在版本库目录下会多了一个叫 .git 的隐藏目录，这个目录是Git来跟踪管理版本库的，没事千万不要手动修改这个目录里面的文件。 也可以使用我们指定目录作为Git仓库。命令是 git init newrepo

$ git init prject;
Initialized empty Git repository in G:/prject/.git/

这个会自动初始化prject目录为版本库，如果prject目录不存在，会自动创建。

添加新文件

版本库创建完成后，可以向版本库中添加新文件，在版本库中新建一个txt文件叫 git.txt ,内容为：

$ git test

通过 git add file 命令可以将新建的文件夹添加至版本库中，

$ git add git.txt;

执行完成之后，没有任何提示，这就表示添加成功了。 git add file1 file2命令也可以用来提交多个文件

$ git add git1.txt git2.txt;

如果需要一次提交所有的文件，命令是

git add .

查看文件状态

输入 git status 可以时刻查看当前版本库的状态。

$ git status;
On branch master

Initial commit

Changes to be committed:
(use “git rm –cached …” to unstage)

new file:   git.txt
new file:   git1.txt
new file:   git2.txt

这里告诉我们有3个新的文件，但是还没有被提交，属于待提交状态。

提交新文件

现在我们已经添加了这些文件，我们希望它们能够真正被保存在Git仓库。

$ git commit -m ‘add new git.txt’;

*** Please tell me who you are.

Run

git config –global user.email “you@example.com“
git config –global user.name “Your Name”

to set your account’s default identity.
Omit –global to set the identity only in this repository.

fatal: unable to auto-detect email address (got ‘Administrator@USER-20161107KU.(none)’)

这里报了一个错误，因为Git是分布式版本控制系统，所以，每个机器都必须自报家门：你的名字和Email地址。需要告诉git你的名字和地址就OK了。

$ git config –global user.name “Your Name”
$ git config –global user.email “email@example.com“

这里 git config命令的--global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和Email地址。 配置完成后继续提交文件，

$ git commit -m ‘add new git.txt’;
[master (root-commit) be0acd0] add new git.txt
3 files changed, 1 insertion(+)
create mode 100644 git.txt
create mode 100644 git1.txt
create mode 100644 git2.txt

提交后可以看见git创建了3个文件; git commit -m后面输入的是本次提交的说明，可以输入任意内容，用来做提交备注或说明，这样你就能从历史记录里方便地找到改动记录。 当我们修改了很多文件，而不想每一个都add，想commit自动来提交本地修改，我们可以使用-a标识。

git commit -a -m “Changed some files”

git commit 命令的-a选项可将所有被修改或者已删除的且已经被git管理的文档提交到仓库中。 可以继续查看版本库状态：

$ git status;
On branch master
nothing to commit, working directory clean

这表示当前版本库、工作区、暂存库的版本是一致的；

小结

初始化一个Git仓库，使用git init命令。 添加文件到Git仓库，分两步：

  • 第一步，使用命令git add ，注意，可反复多次使用，添加多个文件；也可以git add file1 file2一次提交多个文件，还可以 git add . 一次提交改动的所有文件
  • 第二步，使用命令git commit，完成。
